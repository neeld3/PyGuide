Lesson 10: Handling Files and Exceptions in Python

Working with files in Python

Python lets you read from and write to files, such as .txt files. This is useful for saving data, loading settings, logs, or any information you want to keep between runs of your program.

Opening a file

We use the built-in open() function. The second argument is the mode:

  • "r": read (file must exist)
  • "w": write (creates or overwrites the file)
  • "a": append (adds to the end of the file)

Reading from a file

# data.txt must already exist
with open("data.txt", "r") as file:
    content = file.read()
    print(content)

Writing to a file

# This will create or overwrite output.txt
with open("output.txt", "w") as file:
    file.write("Hello from Python!\n")
    file.write("This line will be saved in the file.")

Appending to a file

# This will add to the end of log.txt
with open("log.txt", "a") as file:
    file.write("New log entry\n")

Basic error handling with try/except

Errors (called exceptions) can cause your program to crash. We can use try and except to handle them gracefully. 

try:
    x = 10 / 0
except ZeroDivisionError:
    print("You tried to divide by zero, which is not allowed.")

Common Python errors

1. ZeroDivisionError

Happens when you divide by zero.

try:
    result = 5 / 0
except ZeroDivisionError:
    print("Cannot divide by zero.")

2. TypeError

Happens when you use the wrong type with an operation or function.

try:
    total = "5" + 3   # string + integer is not allowed
except TypeError:
    print("TypeError: You mixed incompatible types.")

3. FileNotFoundError

Happens when you try to open a file that does not exist in the given path.

try:
    with open("missing_file.txt", "r") as file:
        data = file.read()
except FileNotFoundError:
    print("File not found. Please check the file name or path.")

4. ValueError

Happens when a value has the right type, but an invalid format.

user_input = "abc"

try:
    number = int(user_input)   # "abc" cannot be converted to int
except ValueError:
    print("ValueError: Please enter a valid number.")

Combining file handling and error handling

filename = "scores.txt"

try:
    with open(filename, "r") as file:
        for line in file:
            score = int(line.strip())
            print("Score:", score)
except FileNotFoundError:
    print(f"{filename} was not found.")
except ValueError:
    print("Found a line in the file that is not a valid number.")

By combining file operations with proper error handling, your programs become more reliable and user-friendly, even when something goes wrong.
